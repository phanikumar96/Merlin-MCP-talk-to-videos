The Foundation

Meet Marlin-2B

Marlin-2B is a 2-billion parameter video-language model by NemoStation. Unlike image models, Marlin understands the temporal dimension of video — it can watch a clip and tell you what happened, when it happened, and answer questions about specific moments in time.

It ships with two powerful primitives: caption — which produces a full scene description and event timeline — and find — which locates a described event and returns the exact timestamp span.

The Protocol

What is MCP?

Model Context Protocol (MCP) is an open standard that lets AI assistants call external tools. Instead of being limited to text, an AI that speaks MCP can invoke functions — running code, calling APIs, reading files — and get structured results back. Think of it as a universal plugin system for AI.

HTTP Transport

Marlin's MCP server speaks HTTP, so any MCP-compatible client on any machine can call it over the network — no local installation needed on the client side.

🛠️

Typed Tools

Each tool has a typed schema — name, description, and parameter types. The AI reads this schema and knows exactly how to call the tool with the right arguments.

🔒

Stateless & Safe

Every tool call is independent. The server holds no session state between calls, making it simple, predictable, and safe to run in cloud environments.

The Capabilities

Two Powerful Tools

Tool 01
describe_video

Pass any video file path. Marlin watches the entire clip and returns a rich, structured description — a natural-language caption, a scene category, and a timeline of discrete events detected throughout the video.

caption — Full natural-language description of the video
scene — High-level scene category (indoor, outdoor, etc.)
events — Ordered list of detected events with timestamps
describe_video response
{
  "caption": "A man in a blue jacket walks through a busy train station, stops to check his phone, then boards a departing train.",
  "scene": "transportation / public transit",
  "events": [
    "0:00 – Person enters frame from left",
    "0:04 – Subject stops, looks at phone",
    "0:09 – Train arrives at platform",
    "0:14 – Person boards train"
  ]
}
Tool 02
search_video_event

Describe any event in natural language and Marlin scrubs the entire video to find exactly when it occurs. Returns the raw model output, the detected timestamp span, and whether the response was well-formed. Perfect for video search, highlight extraction, and evidence retrieval.

span — Start and end timestamp of the matched event
raw — Raw model response for transparency
format_ok — Whether the span was cleanly parsed
search_video_event response
# query: "person boards the train"
 
{
  "query": "person boards the train",
  "span": {
    "start": 14.2,
    "end": 18.7
  },
  "raw": "The event occurs between 14.2s and 18.7s.",
  "format_ok": true
}
Under the Hood

The Architecture

Two lightweight servers run on the GPU pod. The MCP server translates AI tool calls into HTTP requests to the model API, keeping concerns clean and each layer independently replaceable.

MCP Client
Hermes / Claude
port 8006
marlin_mcp.py
FastMCP Server
port 8005
marlin_api.py
FastAPI Wrapper
NemoStation/Marlin-2B
GPU · bfloat16
marlin_mcp.py

Exposes describe_video and search_video_event as MCP tools. Receives a file path, reads the video, and POSTs it to the API. No model code here.

marlin_api.py

Loads Marlin-2B on startup and holds it in GPU memory. Exposes /describe and /search via FastAPI. Accepts video file uploads, returns JSON.

Set It Up Yourself

The Complete Build Guide

1
Clone the repository

Grab the two server files from GitHub.

bash 
git clone https://github.com/PromptEngineer48/Marlin-2B-MCP.git
cd Marlin-2B-MCP
2
Install Miniconda

We use conda to create an isolated Python environment. Run this on your Linux GPU machine (e.g. RunPod).

bash 
curl -O https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh
bash ./Miniconda3-latest-Linux-x86_64.sh
# Restart your shell or run: source ~/.bashrc
3
Create the conda environment

Python 3.13 in a clean environment named vlm.

bash 
conda create -n vlm python=3.13 -y
conda activate vlm
4
Install MCP and requests

The MCP SDK powers the tool server. Requests handles the HTTP calls from MCP to the model API.

bash 
pip install mcp requests
5
Install the video model stack

Transformers for model loading, PyTorch for inference, torchcodec for video decoding, and supporting libraries.

bash 
pip install "transformers>=5.7.0" "torch>=2.11.0" torchcodec \
            "qwen-vl-utils>=0.0.14" av pillow
6
Install torchvision for your CUDA version

Visit pytorch.org and get the exact install command for your CUDA version. Example for CUDA 12.4:

bash · CUDA 12.4 example 
pip install torchvision --index-url https://download.pytorch.org/whl/cu124
7
Install FastAPI and Uvicorn

FastAPI powers the model API server. Uvicorn is the ASGI server that runs both servers.

bash 
pip install fastapi uvicorn
8
Start both servers

Open two terminals on the GPU machine. Start the model API first and wait for it to finish loading before starting the MCP server.

Terminal 1 — model API · port 8005 
conda activate vlm
uvicorn marlin_api:app --host 0.0.0.0 --port 8005
# Wait for: "Marlin loaded."
Terminal 2 — MCP server · port 8006 
conda activate vlm
uvicorn marlin_mcp:app --host 0.0.0.0 --port 8006
# MCP is now live on :8006
Cloud Deployment

Running on RunPod

RunPod gives you an on-demand GPU with a public HTTPS proxy for any port you expose — perfect for hosting the MCP server so any client can reach it.

🚀

Step 1 — Expose the ports

Before starting the pod, add 8005 and 8006 under HTTP Ports in the RunPod pod configuration. RunPod will provision public HTTPS proxy URLs for both.

:8005 → model API
:8006 → MCP server
🔗

Step 2 — Note your proxy URL

RunPod gives you a public URL in this format. Your MCP clients will use the port 8006 URL to call the tools from anywhere in the world.

your MCP endpoint
https://<pod-id>-8006.proxy.runpod.net/mcp
Client Configuration

Connect to Hermes

Open your Hermes config at C:\Users\<you>\AppData\Local\hermes\config.yaml and add the marlin block under mcp_servers. Replace the URL with your actual RunPod proxy address. Save and restart Hermes.

C:\Users\palas\AppData\Local\hermes\config.yaml 
known_plugin_toolsets:
  cli:
    enabled: true
  mcp_servers:
    claude-code:
      command: claude
      args:
        - mcp
        - serve
      timeout: 300

    marlin:                                         # ← add this block
      url: "https://<your-pod-id>-8006.proxy.runpod.net/mcp"
      timeout: 120
      connect_timeout: 60

  image_gen:
    use_gateway: true
